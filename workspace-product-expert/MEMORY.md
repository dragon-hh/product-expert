# MEMORY.md - 产品专家 AI 长期记忆

本文件是产品专家 agent 的长期业务记忆。OpenClaw 通常会在任务执行时注入长期记忆；只有在需要核对长期规则、数据字典、审核口径或复盘结论时，才需要显式读取本文件。

## 记忆状态标签

- **已确认事实**：来自真实需求、人工审核、正式模板、正式规则或已确认业务数据。
- **默认假设**：当前可用于初稿、初筛或内部测算的假设，外发前必须审核。
- **待专家确认**：缺少依据、存在争议或需要产品/方案/财务/法务确认的内容。
- **历史复盘结论**：由周、月、季、年复盘沉淀的经验，可反向优化流程、模板和模型。

## 平台定位

**已确认事实**：产品专家 AI 是销售增长、方案转化、客户赋能、储能投资决策和经营复盘平台。

它把客户从“模糊需求”推进到“可生成方案、可测算收益、可客户承认、可转订单、可交付、可复盘”的标准流程。

核心闭环：客户信息采集 -> 项目需求明确 -> 应用场景明确 -> 年度需求明确 -> 采购计划明确 -> 储能 ROI 测算 -> 生成《客户需求与项目说明书》 -> 生成标准 Word 技术方案书 -> 方案工程审核 -> 客户赋能 -> 客户承认 -> 报价/合同/订单转化 -> 交付实施 -> FAE 运维 -> 实际运行数据回流 -> ROI 模型校准 -> 管理复盘 -> 反向优化产品、标准包、模板、销售打法和投资模型。

## 核心原则

**已确认事实**：

1. 先需求准入，再生成方案。
2. 先标准模板，再 AI 填充。
3. 先方案工程提交，再客户赋能。
4. 储能项目必须进行 ROI 准入和测算。
5. AI 只做初稿、建议、检查和计算，不替代专业审核。
6. 外发材料必须带审核状态、审核角色和待确认项。
7. 无依据回答必须提示“暂无依据，需专家确认”。

## 准入规则

**已确认事实**：正式方案生成前必须完成以下内容：客户信息、项目需求、应用场景、年度需求、采购计划、风险与限制、储能项目 ROI 前置信息、`客户需求与项目说明书`。

硬性门禁：

- 没有完整客户需求，不生成正式方案。
- 没有应用场景，不匹配标准包。
- 没有年度需求，不生成年度解决方案。
- 没有采购计划，不进入转化预测。
- 没有 ROI 准入数据，不输出正式投资回报结论。
- 没有《客户需求与项目说明书》，不允许生成正式项目技术方案书。

## 需求评分规则

**已确认事实**：

| 维度 | 权重 |
| --- | ---: |
| 客户信息完整度 | 15% |
| 项目需求清晰度 | 20% |
| 应用场景清晰度 | 20% |
| 年度需求清晰度 | 15% |
| 采购计划清晰度 | 15% |
| 风险与限制条件清晰度 | 10% |
| 储能 ROI 数据完整度 | 储能项目附加评分和硬门禁 |

| 分数 | 系统动作 |
| ---: | --- |
| >= 90 | 可生成正式项目技术方案书 |
| 75-89 | 可生成方案初稿，但必须标记待确认项 |
| 60-74 | 不生成正式方案，只输出需求澄清清单和初步建议 |
| < 60 | 不进入方案工程，要求销售补充基础信息 |

## ROI 风控规则

**已确认事实**：储能 ROI 数据不足时，不输出正式投资回报结论；不确定收益必须单列；客户外发前必须由方案工程师和财务或投资测算负责人审核。

收益口径：

- **保守收益**：有明确电价、负荷、运行策略和可审计数据支撑的收益。
- **基础收益**：保守收益加已确认政策补贴或合同可确认收益。
- **增值收益**：需求响应、虚拟电厂、碳减排、扩容延缓、品牌价值等不确定收益，必须单列。

ROI 输出等级：

| 等级 | 使用条件 | 可输出内容 |
| --- | --- | --- |
| 初步测算 | 关键字段不足但可估算范围 | 区间、缺失项、敏感因素，不外发正式结论 |
| 标准测算 | 主要电价、负荷、成本、容量数据齐全 | 回收期、ROI、假设、风险，需审核后外发 |
| 投资决策测算 | 有完整现金流、折现率、运行策略和审核数据 | IRR、NPV、现金流、敏感性，必须人工审核 |

## 方案模板规则

**已确认事实**：方案书必须使用 Word 主模板和模块化章节库。AI 只能填充模板字段、推荐模块和生成草稿内容，不能自由修改目录。

正式项目技术方案书必须经过：需求说明书 -> 模板填充 -> 标准包匹配 -> 配置校验 -> 风险审核 -> 方案工程师审核 -> 正式版归档。

## 权限矩阵

| 角色 | 可做 | 不可自动做 |
| --- | --- | --- |
| 销售 | 提交需求、查看本人客户方案、查看初步 ROI | 审核正式方案、确认技术承诺 |
| 销售主管 | 查看团队需求、方案、客户推进 | 覆盖工程审核结论 |
| 方案工程师 | 审核需求、生成/修改方案、确认技术边界、审核 ROI | 代替法务/财务做最终合规或投资审批 |
| 客户赋能人员 | 生成赋能材料、记录反馈、推进承认 | 承诺技术参数、认证和交付周期 |
| 交付工程师 | 生成实施计划、验收标准、移交资料 | 修改已承认方案商务边界 |
| FAE | 查询资料、修正技术答案、回流运行数据 | 对无依据问题给确定承诺 |
| 产品经理 | 维护产品知识、审核 FAQ、分析竞品问题 | 代替法务审批竞品合规风险 |
| ROI 测算负责人 | 审核 ROI 模型、假设、结果和外发口径 | 跳过方案工程事实校验 |
| 管理层 | 查看全局指标和复盘报告 | 直接修改底层事实记录 |
| 知识管理员 | 维护知识库、标签、版本和有效期 | 篡改审核结论 |
| 系统管理员 | 管理账号、权限、接口和流程 | 代替业务审核 |

## 交付 9 件套规则

**默认假设**：交付 9 件套是已承认方案转交付时的固定资料清单，正式模板和字段可由交付负责人后续确认。

1. 项目实施计划：项目阶段、里程碑、责任人、时间表、关键依赖。
2. 项目任务拆解 WBS：安装、部署、调试、培训、验收、移交任务及责任矩阵。
3. 交付边界确认书：包含项、不包含项、客户配合事项、前置条件、变更触发条件。
4. 配置与物料清单：产品配置、BOM、软件/固件版本、辅材、备件、交付数量。
5. 现场勘查与施工准备清单：场地、电力、网络、并网、消防、环境、安全、入场条件。
6. 安装部署与调试方案：安装步骤、部署拓扑、调试项目、联调方式、回退预案。
7. 验收标准与验收记录表：验收指标、测试方法、通过标准、问题记录和签字栏。
8. 培训与运维移交资料：培训计划、操作手册、运维手册、账号权限、日常巡检。
9. 风险问题与 FAE 移交清单：现场风险、遗留问题、服务 SLA、专家转接、运行数据回流字段。

储能项目必须在交付 9 件套中补充充放电策略、EMS 参数、监控要求、收益数据采集字段和安全风险。
