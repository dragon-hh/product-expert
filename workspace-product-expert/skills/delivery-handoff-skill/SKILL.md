# 交付移交资料

## Inputs

- 已承认方案。
- 订单信息、采购计划、配置清单。
- 交付边界、验收要求、现场条件。
- 交付团队、FAE、售后和客户侧责任人。
- 储能项目的运行策略要求、EMS 参数、监控要求。

## Input Sources

输入不要求用户一次性手工填写完整。按以下优先级取数：

1. **用户本次输入**：如果用户直接提供已承认方案、订单信息、配置清单或现场条件，以本次输入为最高优先级。
2. **本地数据目录**：默认读取 `../../data/catalog.json`，再按 catalog 指向的 JSON 表取数。
3. **方案主表**：从 `../../data/solution_master.json` 按 `solution_id`、`demand_id`、客户名称或项目名称读取方案状态、客户承认状态、配置清单、交付边界、验收要求、方案附件。
4. **订单/商机主表**：从 `../../data/order_master.json` 按 `order_id`、`opportunity_id` 或客户项目匹配订单金额、订单状态、采购计划、合同节点、交付范围。
5. **文档索引与附件**：从 `../../data/document_library.json` 找到已审核方案、客户确认页、需求说明书，再提取技术边界、交付边界、验收标准、储能运行策略要求。
6. **交付项目表**：从 `../../data/delivery_project.json` 读取交付负责人、FAE、售后负责人、客户侧联系人和责任矩阵。
7. **现场勘查表**：从 `../../data/site_survey.json` 读取场地、电力、网络、并网、消防、环境、安全、入场条件。
8. **补充导入**：如果交付团队、销售或客户提供了新的订单、现场或责任人信息，先按 `../../references/writeback-contract.md` 归一化为本地 GDP 写回候选。
9. **用户补充**：以上来源仍缺失时，输出缺失字段和补采问题，不得自行补造。

最小可执行输入是 `solution_id` 或 `order_id`。如果两者都没有，且用户没有贴方案或订单内容，返回 `blocked`。
如果本地 JSON 表存在但 `records` 为空，视为尚未导入真实业务数据，应输出缺失数据源，而不是生成正式交付资料。

## Procedure

1. 确认客户承认状态、订单信息、采购计划、配置清单和交付边界是否一致。
2. 按 Input Sources 补齐字段，并保留每个字段的来源。
3. 生成固定“交付 9 件套”，不得只写统称：
   - `01-项目实施计划`：项目阶段、里程碑、责任人、时间表、关键依赖。
   - `02-项目任务拆解 WBS`：安装、部署、调试、培训、验收、移交任务及责任矩阵。
   - `03-交付边界确认书`：包含项、不包含项、客户配合事项、前置条件、变更触发条件。
   - `04-配置与物料清单`：产品配置、BOM、软件/固件版本、辅材、备件、交付数量。
   - `05-现场勘查与施工准备清单`：场地、电力、网络、并网、消防、环境、安全、入场条件。
   - `06-安装部署与调试方案`：安装步骤、部署拓扑、调试项目、联调方式、回退预案。
   - `07-验收标准与验收记录表`：验收指标、测试方法、通过标准、问题记录和签字栏。
   - `08-培训与运维移交资料`：培训计划、操作手册、运维手册、账号权限、日常巡检。
   - `09-风险问题与 FAE 移交清单`：现场风险、遗留问题、服务 SLA、专家转接、运行数据回流字段。
4. 明确每份资料的输入来源、输出格式、审核角色和是否可外发。
5. 储能项目必须在第 06、07、08、09 项中补充充放电策略、EMS 参数、监控要求、收益数据采集字段和安全风险。
6. 输出项目移交摘要、遗留问题、运行数据字段和待确认风险。

## Outputs

- 交付 9 件套完整清单和正文草稿。
- 项目移交摘要。
- 储能运行策略和运行数据采集字段。
- 现场风险提示和遗留问题清单。
- FAE/售后移交记录。
- 审核角色、审核状态和待确认事项。
- `writeback_plan`：目标表 `delivery_project`、`document_library`、`site_survey`，登记交付 9 件套路径、交付边界、验收要求、责任人和运行数据回流字段。

## Error Handling

- 没有 `solution_id`、`order_id`、方案正文或订单内容时，返回 `blocked`。
- 未承认方案不得进入正式交付移交。
- 订单和方案不一致时返回 `blocked`，列出冲突字段。
- 交付负责人、客户侧责任人或验收要求缺失时返回 `needs_delivery_input`。
- 储能运行策略、EMS 参数或监控要求缺失时返回 `needs_solution_review`。
