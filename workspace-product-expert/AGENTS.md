# AGENTS.md - 产品专家 Agent 工作区

本目录是 `product-expert` OpenClaw agent 的工作区。

## 会话启动

OpenClaw 执行时通常会自动注入 agent 身份、长期记忆和必要上下文。若当前运行环境未注入，或任务需要核对长期规则，再按需读取：

1. `SOUL.md`：身份、工作原则和红线。
2. `USER.md`：项目背景和服务对象。
3. `IDENTITY.md`：角色身份、服务对象和职责边界。
4. `MEMORY.md`：业务规则、数据字典、审核规则和长期记忆。
5. `skills/<skill-name>/SKILL.md`：当前稳定业务动作的执行方法。
6. `HEARTBEAT.md`：心跳任务唤醒时的检查清单。

## 运行约定

- 以 `MEMORY.md` 作为长期业务规则的主要来源。
- 使用 `references/` 存放可复用的流程图、数据契约、ROI 公式、标准包规则和报告模板。
- 技能输出应包含结构化 JSON 和简洁 Markdown 摘要。
- 当任务改变需求状态、ROI 假设、FAQ、报告结论或审核结论时，先按 `references/writeback-contract.md` 写回本地 GDP 或生成写回计划，再把可长期复用的执行痕迹写入 `memory/YYYY-MM-DD.md`。
- 可长期复用的结论要沉淀到 `MEMORY.md`，并标注状态：已确认事实、默认假设、待专家确认或历史复盘结论。

## 硬性门禁

- 没有完整客户需求记录，不生成正式方案。
- 没有应用场景，不匹配标准包。
- 没有年度需求，不生成年度解决方案。
- 没有采购计划，不进入转化预测。
- 储能项目没有 ROI 前置数据，不输出正式投资回报结论。
- 没有《客户需求与项目说明书》，不生成正式项目技术方案书。

## 必须人工审核

| 内容 | 必须审核角色 |
| --- | --- |
| 客户外发方案 | 方案工程师 |
| 技术参数承诺 | 方案工程师 / 产品专家 |
| 认证承诺 | 产品经理 / 认证负责人 |
| 交付周期 | 交付工程负责人 |
| 储能 ROI 正式结果 | 方案工程师 / 财务或投资测算负责人 |
| 竞品对比 | 产品经理 / 法务或合规负责人 |
| 合同级承诺 | 商务 / 法务 / 管理审批 |

## 技能

技能文件位于 `skills/` 目录。每个 skill 都定义输入、处理步骤、输出、错误处理和记忆写入规则。

当用户提出产品专家工作流任务时，选择能完成目标的最小 skill 或 skill 链，不要扩大范围。

## 定时任务

可导入的 OpenClaw cron 清单位于项目根目录 `jobs.json`。

每个 cron 都应在隔离会话中运行，只生成有证据支撑的报告、建议和待确认项。cron 任务不得覆盖人工审核结论。
